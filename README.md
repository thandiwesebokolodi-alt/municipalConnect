📌 Civic Service Reporting System (HCI Project)

This project is a Human-Computer Interaction (HCI) design solution aimed at improving how citizens report and track community service issues such as potholes, water leaks, and electricity failures.

🚀 Overview

The system enhances existing platforms like RequestorApp by focusing on usability, transparency, and user empowerment. It provides a more reliable and user-friendly way for citizens to communicate with municipalities.

✨ Key Features
📍 Interactive Map Dashboard – View and track issues in your community using color-coded pins
🔄 One-Click Reopen – Easily reopen unresolved reports without starting over
📸 Photo Verification – Upload before/after images to confirm issue resolution
🔔 Real-Time Notifications – Get updates via SMS, email, or app notifications
👥 Community Support – “Add your voice” to existing reports
⭐ Feedback & Ratings – Rate service delivery and improve accountability
🎯 Objectives
Improve effectiveness (issues are actually resolved)
Increase efficiency (faster, simpler reporting process)
Enhance safety & control (users can verify and dispute outcomes)
🧠 Design Approach

The system was developed using:

User research (personas & interviews)
Usability goals (effectiveness, efficiency, safety)
Design principles (visibility, feedback, consistency)
AI-supported evaluation and iterative improvement
🛠️ Technologies (Conceptual / Prototype)
Django (web framework)
HTML, CSS (Tailwind)
JavaScript (interactive UI elements)
📖 Purpose

This project demonstrates how user-centered design and AI-assisted evaluation can be used to create more inclusive, transparent, and effective digital systems for public service delivery.
